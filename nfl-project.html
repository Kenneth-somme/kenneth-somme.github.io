Winning by the numbers

Turnovers, Penalties & Sacks in the NFL (2002-2025)

SQL PostgreSQL Power BI

Project Overview

This project examines what statistically separates winning teams from losing teams in the NFL. Using team-level game data from 2002 to 2025, I analyzed how interception differential, sack differential, and penalty behavior characterize winning and losing seasons — and whether those patterns have remained consistent across three distinct eras of professional football.

The project involved querying a PostgreSQL database with complex CTEs and aggregations, and presenting the findings through an interactive Power BI dashboard.

Objective

To describe the statistical profile of winning and losing NFL teams across 23 seasons — identifying which metrics most strongly separate the two groups, and examining whether those characteristics have shifted across the Traditional Era (2002–2010), Passing Revolution Era (2011–2017), and Modern Analytics Era (2018–2025).

Features

Era Breakdown: Winning team metrics tracked across three eras, comparing average wins, pass yards per attempt, and INT differential — revealing that the gap between winners and losers has remained remarkably stable despite the league's evolution.

INT Differential Over Time: A season-by-season line chart showing the consistent separation between winning and losing teams in interception differential from 2002 to 2025.

Scatter Plot Analysis: Three correlation charts examining how INT differential, sack differential, and penalty amount each relate to team wins — allowing direct visual comparison of which factors matter most.

KPI Cards: At-a-glance summary metrics highlighting the average penalty yards and INT differential for both winning and losing teams across the full dataset.

Interactive Filters: Dashboard sliceable by team and season for deeper exploration.

SQL Analysis Queries

Query 1 — This query transforms raw game-by-game data into a clean, team-season summary ready for analysis. A UNION ALL flips the dataset from game rows into team rows — giving both the home and away team their own record per game with correct win/loss assignments. From there, three CTEs handle filtering (playoffs removed), aggregation (season totals per team), and calculation of derived metrics like interception differential, pass yards per attempt, and sack differential. The final output classifies each team-season as Winning or Losing based on win percentage.


                WITH win_loss AS (
                	SELECT
                		season,
                		week,
                		home AS team,
                		CASE
                			WHEN score_home > score_away THEN 1
                			WHEN score_home < score_away THEN 0
                			ELSE 0.5
                		END AS w_l,
                		pen_num_home AS pen_num,
                		pen_yards_home AS pen_yards,
                		interceptions_away AS interceptions_for,
                		interceptions_home AS interceptions_against,
                		sacks_num_away AS sacks_forced,
                		sacks_num_home AS sacks_recieved,
                		pass_att_home AS pass_att,
                		pass_comp_home AS pass_comp,
                		pass_yards_home AS pass_yards,
                		rush_att_home AS rush_att,
                		rush_yards_home AS rush_yards,
                		score_home AS points_for,
                		score_away AS points_against,
                		sacks_yards_home AS sack_yards
                	FROM nfl_game_stats
                	UNION ALL
                	SELECT
                		season,
                		week,
                		away,
                		CASE
                			WHEN score_away > score_home THEN 1
                			WHEN score_away < score_home THEN 0
                			ELSE 0.5
                		END,
                		pen_num_away,
                		pen_yards_away,
                		interceptions_home,
                		interceptions_away,
                		sacks_num_home,
                		sacks_num_away,
                		pass_att_away,
                		pass_comp_away,
                		pass_yards_away,
                		rush_att_away,
                		rush_yards_away,
                		score_away,
                		score_home,
                		sacks_yards_away
                	FROM nfl_game_stats
                ), sorting AS(
                	SELECT *
                	FROM win_loss
                	WHERE week NOT IN ('Conference', 'Wildcard', 'Superbowl', 'Division')
                ), aggre AS (
                	SELECT
                		season,
                		team,
                		COUNT(*) AS games,
                		SUM(w_l) AS w_l,
                		SUM(pen_num) AS pen_num,
                		SUM(pen_yards) AS pen_yards,
                		SUM(interceptions_for) AS ints_for,
                		SUM(interceptions_against) AS ints_against,
                		SUM(pass_yards) AS pass_yards,
                		SUM(pass_att) AS pass_att,
                		SUM(pass_comp) AS pass_comp,
                		SUM(rush_yards) AS rush_yards,
                		SUM(rush_att) AS rush_att,
                		SUM(sack_yards) AS sack_yards,
                		SUM(sacks_forced) AS sacks_forced,
                		SUM(sacks_recieved) AS sacks_recieved,
                		SUM(points_for) AS points_for,
                		SUM(points_against) AS points_against
                	FROM sorting
                	GROUP BY season, team
                ), cleaning AS(
                	SELECT 
                		season,
                		team,
                		w_l AS wins,
                		ROUND((w_l) / games, 3) AS w_l_pct,
                		pen_num,
                		pen_yards,
                		ints_for - ints_against AS interception_differential,
                		ROUND((pass_yards::NUMERIC + sack_yards) / pass_att, 1) AS pass_yards_per_attempt,
                		ROUND(((pass_comp::NUMERIC / pass_att) * 100), 1)  AS pass_completion_percent,
                		ROUND(rush_yards::NUMERIC / rush_att, 1) AS rush_yards_per_attempt,
                		sacks_forced - sacks_recieved AS sacks_differential,
                		points_for,
                		points_against
                	FROM aggre
                )
                SELECT *,
                	CASE
                		WHEN w_l_pct > 0.500 THEN 'Winning'
                		ELSE 'Losing'
                	END AS team_record
                FROM cleaning
                ORDER BY w_l_pct DESC;

Results

The data reveals a strong and consistent correlation between both INT differential and sack differential and team wins. Teams with a positive INT differential — throwing fewer interceptions than their opponents — win significantly more games, with very few outliers across the full 23-season period. The same pattern holds for sack differential: teams that pressure the quarterback more than they are pressured tend to win more.

Perhaps the most unexpected finding was in penalty behavior. Despite common coaching emphasis on discipline, penalty amount shows no meaningful correlation with wins. Teams with high penalty counts win just as often as disciplined ones — suggesting that penalty volume is not a reliable predictor of team success.

The era comparison reinforces how stable these patterns are. While the passing game has evolved significantly, the fundamental indicators of winning — protecting the ball and generating pressure — have remained constant across all three eras.

Recommendation: For teams focused on competitive improvement, the data points clearly toward turnover margin and pass rush as the highest-leverage areas. Penalty reduction, while valuable for other reasons, does not appear to be a significant driver of winning.

Project Structure

Data — Raw CSV file containing NFL team game statistics from 2002 to 2025, sourced from Kaggle.
SQL / PostgreSQL — Database built in PostgreSQL 17. Queries written using CTEs, aggregations, and subqueries to extract, classify, and transform data across winning and losing game outcomes.
Power BI — Interactive dashboard featuring KPI cards, scatter plots, a trend line chart, and era-based bar charts with team and season filters.
